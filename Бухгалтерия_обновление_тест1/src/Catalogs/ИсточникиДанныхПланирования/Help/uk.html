Джерела даних планування

Джерела даних планування використовуються для заповнення планів продажів, планів продажів по категоріях, планів купівель, планів збирання (розбирання) і планів виробництва.

В якості джерел даних можуть бути використані будь-які дані, що зберігаються в інформаційній базі. Для настройки джерел даних планування використовується схема компонування даних. У програмі також передбачені різні варіанти шаблонів, за допомогою яких можна швидко зробити настройки джерел даних. Настройка джерел даних проводиться в окремому діалоговому вікні, який відкривається по кнопці Редагувати у формі довідника.

Приклад. Як джерело даних при плануванні можуть бути використані дані про продажі або закупівлі товарів за попередні періоди, дані про поточні замовлення клієнтів, передбачувані постачання товарів і т. д.

Опис можливостей шаблонів джерел
  • Змінити на сезонний коефіцієнт – дозволяє скоригувати обрані дані на сезонний коефіцієнт. Коефіцієнт, на який множиться результат джерела, розраховується як відношення сезонного коефіцієнта за період планування (період документа плану) до сезонного коефіцієнта за період вибірки даних джерела;
  • Параметри забезпечення - дозволяє відфільтрувати результат вибірки джерела за параметрами забезпечення або за основним способом забезпечення потреб;
  • Асортимент - дозволяє відфільтрувати результат вибірки джерела за настройками асортименту (товар, дозволений до купівель або продажу, ролі асортименту, формат магазину і т. д.). У правилі заповнення документа в настройках джерела необхідно в параметрі "Асортимент на дату" вказати дату, на яку визначаються настройки асортименту і настройки форматів магазинів для складів.
  • Вибірка зі зміщенням періоду - дозволяє вказати кількість періодів зміщення відносно періоду планування в документі. якщо документ план складено з періодичністю "Місяць" на період з Січня 2015 по Березень 2015 (три місяці), то вказавши в правилі заповнення документа параметр "зі зміщенням періоду" = -3 - джерело вибере дані за період з Жовтня 2014 по Грудень 2014. У загальному випадку, джерело з можливістю вибірки зі зміщенням періоду можна використовувати для вибірки "за період" - вказуючи в правилі заповнення документа плану, фіксований період.
  • Відбір ABCXYZ - дозволяє відібрати товар за класифікацією ABC і XYZ, наприклад: у відборі можна вказати "Номенклатура.ABC (валовий прибуток) = B-клас".

  • Замовлення на переміщення (відвантаження) - вибирає з документів Замовлення на переміщення товари, які заплановані до відвантаження зі складу. Можливості: Асортимент, Параметри забезпечення, зміна на сезонний коефіцієнт, вибірка зі зміщенням періоду, відбір ABCXYZ.

  • Збирання ( розбирання) - вибирає обороти документів збирання (розбирання) по регістру накопичення "Партії товарів організацій", по товарах які надійшли на склад. Можливості: Асортимент, Параметри забезпечення, зміна на сезонний коефіцієнт, вибірка зі зміщенням періоду, відбір ABCXYZ.

Настройки джерела даних

Для кожного джерела даних вводиться інформація про те, як буде відбуватися вибірка даних по даному джерелу:

  • за період - при використанні джерела в правилі заповнення документа плану необхідно буде вказати період дат. Джерело (схема компонування) повинно підтримувати параметри НачалоПериода і КонецПериода.
  • на дату - в правилі заповнення необхідно буде вказати дату, на яку вибираються дані. Джерело даних повинно підтримувати параметр Период.
  • без обмеження - в джерело не будуть автоматично передаватися параметри періоду.
  • зі зміщенням періоду - в правилі заповнення в настройках джерела необхідно вказати кількість періодів зміщення щодо періоду планування. Джерело даних повинно підтримувати параметри: СмещениеПериода, НачалоПериодаСмещения, КонецПериодаСмещения, Периодичность.

Шаблони джерел, які обирають дані за період, можуть бути використані для зазначення певного періоду (за період) або зі зміщенням від періоду зазначеного в документі плану (зі зміщенням).

Для джерела можна також ввести певні обмеження (відбори даних). При цьому розрізняються два види відборів: напередвизначений відбір даних і користувацькі настройки відбору даних, які будуть доступні користувачеві при формуванні планів продажів і внутрішнього споживання.

Приклад Менеджери повинні прогнозувати продажі різних видів товарів у певному бізнес-регіоні. При плануванні продажів вони повинні аналізувати продажі товарів тільки в цьому регіоні, але при цьому складати окремі плани продажів за кожним видом номенклатури. В цьому випадку, в якості напередвизначеного відбору для джерела даних Продажі в регіоні буде відбір за продажами в зазначеному бізнес-регіоні (Партнер.Бизнес регион равно ...), а в якості користувацької настройки буде вказана можливість встановлення відбору за довідником Номенклатура.

Приклад. Необхідно планувати продажі на наступний місяць за максимумом останніх 3-х місяців. Для цього в правило заповнення документа плану додати групу Максимум і включити в неї 3 джерела створені з шаблону Продажі, встановивши що Дані вибираються: зі зміщенням періоду. Натиснувши кнопку "Редагувати" у формі джерела вказати для параметра Зміщення періоду значення "-1", "-2" і " -3 " для кожного джерела відповідно. При плануванні продажів наступного місяця джерела оберуть дані поточного місяця, минулого і позаминулого.

Створення довільного джерела даних планування

Довільне джерело можна створити наступними способами:

  • Створивши нове джерело даних планування (схема компонування даних - Довільна) і натиснувши кнопку Редагувати, у формі настройок натиснувши кнопку Редагувати схему компонування - далі створити запит та всі необхідні настройки схеми.
  • На основі будь-якого з існуючих шаблонів - у формі джерела даних вибрати будь-який шаблон, близький за настройками, натиснути кнопки Редагувати - Редагувати схему компонування і внести зміни в текст запиту або в інші настройки схеми. Назва схеми буде замінена на значення "Довільна".

У тексті запиту довільного джерела можна використовувати параметри, що заповнюються за реквізитами документа плану:
&Ссылка, &Подразделение, &Склад, &ФорматМагазина, &Партнер, &Соглашение, &Статус, &Сценарий, &Периодичность, &Валюта, &НачалоПериода, &ОкончаниеПериода, &Ответственный, &Менеджер, &ВидПлана.

Завантаження довільної схеми компонування даних

Для завантаження довільної схеми компонування даних (СКД) з файлу необхідно натиснути кнопку Редагувати і вибрати пункт Завантажити схему з файлу. Якщо при цьому був обраний шаблон схеми, наприклад "План продажів", то він буде замінений на "Довільна".

Для джерел повинна бути визначена схема їх спільної взаємодії (вибір максимального з джерел, складання джерел тощо). Це правило заповнення документа плану може бути зазначено в самому документі або у виді плану як правило по умовчанню.
